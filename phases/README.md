## Phases

This folder organizes work **phase-by-phase**. Each phase directory contains:

- A short `README.md` summarizing what the phase delivers.
- A `docs/` bundle with the phase's architecture spec (copied for convenience).
- A `deliverables.md` manifest listing the exact code files created/changed for that phase (in their runtime locations like `backend/`).

### Index

- `phase-1/` – Requirements & high-level architecture (foundation + skeleton backend)
- `phase-2/` – Data ingestion & preprocessing (dataset loader + normalization + indices)
- `phase-3/` – Core filtering engine (deterministic filtering + heuristic pre-ranking)
- `phase-4/` – LLM integration & prompt orchestration
- `phase-5/` – Backend API layer
- `phase-6/` – Frontend/UI layer
- `phase-7/` – Observability & evaluation
- `phase-8/` – Scaling & extensions (optional)
